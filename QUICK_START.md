# 对话文件上传功能 - 快速开始指南

## 🚀 立即开始使用

### 步骤 1: 拉取最新代码

```bash
git pull origin main
```

### 步骤 2: 启动服务

#### 启动后端 (Spring Boot)
```bash
cd backend
# 如果使用 Maven
./mvnw spring-boot:run

# 或者使用 IDE 运行 Application.java
```

后端将在 http://localhost:8080 启动

#### 启动前端 (Vue3)
```bash
cd frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

前端将在 http://localhost:3000 启动

### 步骤 3: 登录系统

1. 打开浏览器访问 http://localhost:3000
2. 使用您的账号登录
3. 确保您有 API 使用权限

### 步骤 4: 开始使用文件上传

#### 方式一: 点击上传

1. 进入"对话"页面
2. 选择一个 AI 模型
3. 点击"上传文件"按钮
4. 选择一个或多个文本文件（最多10个，每个不超过10MB）
5. 等待上传完成（会显示文件标签）
6. 可选：输入文本消息
7. 点击"发送"按钮

#### 方式二: 拖拽上传

1. 进入"对话"页面
2. 选择一个 AI 模型
3. 直接拖拽文件到上传按钮区域
4. 等待上传完成
5. 可选：输入文本消息
6. 点击"发送"按钮

## 📁 支持的文件类型

### 纯文本文件
- `.txt` - 纯文本
- `.md` - Markdown 文档
- `.log` - 日志文件

### 配置文件
- `.json` - JSON 配置
- `.xml` - XML 文件
- `.yml` / `.yaml` - YAML 配置
- `.properties` - Properties 配置
- `.csv` - CSV 数据

### 代码文件
- `.java` - Java 源代码
- `.py` - Python 脚本
- `.js` - JavaScript
- `.ts` - TypeScript
- `.html` - HTML
- `.css` - CSS 样式

## 💡 使用示例

### 示例 1: 代码审查

**上传文件**: `MyClass.java`

**提问**: "这段代码有什么问题？如何优化？"

**AI 响应**: 基于文件内容分析代码质量、发现问题、提供优化建议

### 示例 2: 配置检查

**上传文件**: `application.yml`

**提问**: "这个配置文件是否正确？有什么需要注意的地方？"

**AI 响应**: 验证配置格式、解释各项配置、指出潜在问题

### 示例 3: 日志分析

**上传文件**: `error.log`

**提问**: "从这个日志文件中找出所有错误，并提供解决方案"

**AI 响应**: 识别错误模式、分析根本原因、提供排查建议

### 示例 4: 多文件对比

**上传文件**: `version1.py`, `version2.py`

**提问**: "对比这两个文件的差异"

**AI 响应**: 识别代码变更、分析影响、评估改进

## ⚡ 高级技巧

### 技巧 1: 批量上传
一次性上传多个相关文件，AI 可以综合分析所有内容

### 技巧 2: 组合查询
上传文件后继续在同一会话中提问，AI 会记住文件内容

### 技巧 3: 精准提问
明确指出您关注的部分，如"重点分析第50-100行"

### 技巧 4: 增量分析
先上传主文件，再根据 AI 反馈上传相关文件

## 🔍 故障排查

### 问题 1: 文件上传失败

**可能原因**:
- 文件超过 10MB
- 文件类型不支持
- 网络连接问题

**解决方案**:
1. 检查文件大小
2. 确认文件扩展名在支持列表中
3. 检查网络连接

### 问题 2: 看不到上传按钮

**可能原因**:
- 未登录
- 没有 API 使用权限
- 浏览器缓存问题

**解决方案**:
1. 确保已登录
2. 联系管理员开通权限
3. 清除浏览器缓存并刷新

### 问题 3: 上传后无法发送

**可能原因**:
- 未选择 AI 模型
- 文件内容为空
- 超出配额限制

**解决方案**:
1. 在页面顶部选择 AI 模型
2. 检查文件是否有内容
3. 检查 API 使用配额

## 📊 限制说明

### 文件限制
- **单文件大小**: 最大 10MB
- **文件数量**: 每次最多 10 个
- **文本内容**: 提取最多 100KB（超出部分会被截断）

### 性能考虑
- 大文件会消耗更多时间解析
- 多文件会增加 AI 处理时间
- 超长内容会消耗更多 token

## 🎯 最佳实践

### 1. 文件准备
- 确保文件使用 UTF-8 编码
- 移除不必要的注释和空行
- 将大文件拆分为小文件

### 2. 提问技巧
- 明确您的需求
- 指定关注的部分
- 提供必要的上下文

### 3. 效率优化
- 只上传相关文件
- 避免重复上传
- 充分利用会话历史

### 4. 隐私保护
- 不要上传包含敏感信息的文件
- 上传前检查文件内容
- 使用完毕后删除会话
